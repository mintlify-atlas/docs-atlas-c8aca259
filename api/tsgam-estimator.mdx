title: 'TsgamEstimator'
description: 'The main estimator class implementing the scikit-learn fit/predict API'
---

`TsgamEstimator` fits a Time Series Generalized Additive Model (TSGAM) for time series forecasting. It combines multi-periodic Fourier basis functions for seasonal patterns, cubic spline or linear basis functions for exogenous variables with lead/lag support, an optional trend term, an optional outlier detector, and an optional autoregressive residual model. Coefficients are fitted via regularized optimization using CVXPY.

## Constructor

```python
TsgamEstimator(config: TsgamEstimatorConfig)
```

<ParamField path="config" type="TsgamEstimatorConfig" required>
  All model settings. See [TsgamEstimatorConfig](/api/tsgam-estimator-config) for the full list of options.
</ParamField>

## Methods

### fit

```python
fit(X, y, sample_weight=None) -> TsgamEstimator
```

Fit the TSGAM model to training data.

The method extracts and validates timestamps from `X`, builds Fourier and exogenous basis matrices, solves the regularized optimization problem, and optionally fits an AR model on residuals.

**Parameters**

<ParamField path="X" type="DataFrame" required>
  Training data with exogenous variables. Must have a `DatetimeIndex`, or the first column must contain datetime values. Remaining columns are the exogenous variables (for example, temperature). Column order must match the order in `config.exog_config`.

  Gaps in the training timestamps are allowed — the method infers the base frequency from time differences and masks samples with NaN values introduced by lead/lag operations.
</ParamField>

<ParamField path="y" type="array-like of shape (n_samples,)" required>
  Target values. Can be in any scale, though log transformation is common when model components are multiplicative. Must not contain NaN.
</ParamField>

<ParamField path="sample_weight" type="array-like of shape (n_samples,)" default="None">
  Optional sample weights for weighted least squares. Values must be non-negative and the array must match the length of `y`. If `None`, all samples are weighted equally. The AR step is always unweighted.
</ParamField>

**Returns**

<ResponseField name="self" type="TsgamEstimator">
  Returns the fitted estimator to allow method chaining.
</ResponseField>

**Raises**

- `ValueError` — if `X` does not have a proper timestamp index or column
- `ValueError` — if `sort_index=False` and the index is not pre-sorted chronologically
- `ValueError` — if `sample_weight` has the wrong shape or contains negative values
- `ValueError` — if there are insufficient samples for the configured lags

---

### predict

```python
predict(X, remove_periodic=False, remove_exogenous=False, remove_trend=False) -> ndarray
```

Predict target values for new data using the fitted model components: the constant term, Fourier seasonal patterns, exogenous variable terms, and trend (if configured). The AR model is **not** included in predictions — use `sample()` for AR noise rollout.

<Note>
  Prediction timestamps must be at the same frequency as training data. Always use a regularly spaced date range for prediction.
</Note>

**Parameters**

<ParamField path="X" type="DataFrame" required>
  Input data with exogenous variables. Must have a `DatetimeIndex` or a datetime first column. Must use the same frequency as the training data. Column order must match the training data.
</ParamField>

<ParamField path="remove_periodic" type="bool" default="False">
  If `True`, the Fourier seasonal component is excluded from the prediction. Useful for ablation studies or isolating the contribution of other components.
</ParamField>

<ParamField path="remove_exogenous" type="bool" default="False">
  If `True`, all exogenous variable contributions are excluded from the prediction.
</ParamField>

<ParamField path="remove_trend" type="bool" default="False">
  If `True`, the trend component is excluded from the prediction.
</ParamField>

**Returns**

<ResponseField name="predictions" type="ndarray of shape (n_samples,)">
  Predicted values in the same scale as the training data. If the training data was log-transformed, predictions are in log space and can be converted back with `np.exp(predictions)`.
</ResponseField>

**Raises**

- `ValueError` — if the model is not fitted (call `fit` first)
- `ValueError` — if `X` does not have a proper timestamp index or column
- `ValueError` — if the frequency does not match the training frequency

## Fitted attributes

These attributes are set after calling `fit`.

<Warning>
  After fitting, check `problem_.status` to verify the solver converged. Accepted values are `"optimal"` and `"optimal_inaccurate"`. A status outside these values raises a `ValueError` during `fit`, but checking the status yourself is useful when diagnosing borderline problems.
</Warning>

<ResponseField name="problem_" type="cvxpy.Problem">
  The solved CVXPY optimization problem. Inspect `problem_.status` (expect `"optimal"` or `"optimal_inaccurate"`) and `problem_.value` for the final objective.
</ResponseField>

<ResponseField name="freq_" type="str">
  Inferred frequency string of the training time series, stored with a numeric prefix and in lowercase (for example, `'1h'` for hourly data, `'1D'` for daily).
</ResponseField>

<ResponseField name="time_reference_" type="Timestamp">
  Reference timestamp used for phase alignment — the first timestamp from the training data. All numeric time indices are measured in hours since this reference.
</ResponseField>

<ResponseField name="time_indices_" type="ndarray">
  Numeric time indices (hours since `time_reference_`) used during `fit`.
</ResponseField>

<ResponseField name="variables_" type="dict">
  Dictionary of CVXPY variables containing fitted coefficients. Keys present depend on which components were configured:

  - `'constant'` — scalar intercept term (always present)
  - `'fourier_coef'` — Fourier coefficients (present when `multi_periodic_config` is set)
  - `'exog_coef_{i}'` — coefficient matrix for exogenous variable `i`, shape `(basis_dim, num_lags)` (present for each entry in `exog_config`)
  - `'trend'` — per-period trend values (present when `trend_config` is set and `trend_type != 'none'`)
  - `'trend_slope'` — scalar slope (present when `trend_type='linear'`)
  - `'outlier'` — per-period sparse outlier corrections (present when `outlier_config` is set)

  Access the numeric value of any variable with `.value`, for example `estimator.variables_['constant'].value`.
</ResponseField>

<ResponseField name="exog_knots_" type="list">
  Knot locations auto-computed during `fit` for spline exogenous variables. One entry per variable in `exog_config`: a `ndarray` of knot positions when knots were auto-generated, or `None` when knots were supplied via `TsgamSplineConfig.knots` or the variable uses `TsgamLinearConfig`. Reused during `predict` to ensure consistent basis matrices.
</ResponseField>

<ResponseField name="combined_valid_mask_" type="ndarray">
  Boolean mask of shape `(n_samples,)` indicating valid training samples — samples that do not have NaN values introduced by lead/lag operations on exogenous variables.
</ResponseField>

<ResponseField name="ar_coef_" type="ndarray or None">
  Fitted AR coefficients of shape `(n_lags,)`. `None` when `ar_config` is not set or when the AR optimization did not converge.
</ResponseField>

<ResponseField name="ar_intercept_" type="float or None">
  Fitted AR intercept. `None` when `ar_config` is not set or the AR model did not converge.
</ResponseField>

<ResponseField name="ar_noise_loc_" type="float or None">
  Location parameter of the Laplace noise distribution fitted to AR residuals. Used by `sample()`. `None` when no AR model is fitted.
</ResponseField>

<ResponseField name="ar_noise_scale_" type="float or None">
  Scale parameter of the Laplace noise distribution fitted to AR residuals. Used by `sample()`. `None` when no AR model is fitted.
</ResponseField>

## Example

```python
import numpy as np
import pandas as pd
from tsgam_estimator import (
    TsgamEstimator,
    TsgamEstimatorConfig,
    TsgamMultiPeriodicConfig,
    TsgamSplineConfig,
    TsgamOutlierConfig,
)

# Build configuration
multi_periodic = TsgamMultiPeriodicConfig(
    num_harmonics=[6, 4, 3],
    periods=[365.2425 * 24, 7 * 24, 24],  # yearly, weekly, daily
)
exog_config = [TsgamSplineConfig(n_knots=10, lags=[-1, 0, 1])]
outlier_config = TsgamOutlierConfig(reg_weight=0.002)

config = TsgamEstimatorConfig(
    multi_periodic_config=multi_periodic,
    exog_config=exog_config,
    outlier_config=outlier_config,
)

# Create estimator
estimator = TsgamEstimator(config=config)

# Prepare training data — X must be a DataFrame with DatetimeIndex
dates = pd.date_range("2020-01-01", periods=1000, freq="h")
X = pd.DataFrame({"temp": np.random.randn(1000)}, index=dates)
y = np.log(np.random.rand(1000) * 100 + 50)  # log-transform recommended

# Fit
estimator.fit(X, y)
print(f"Problem status: {estimator.problem_.status}")

# Inspect detected outliers
outlier_values = estimator.variables_["outlier"].value
print(f"Detected {np.sum(np.abs(outlier_values) > 0.1)} outlier days")

# Predict on new data
X_pred = pd.DataFrame(
    {"temp": np.random.randn(100)},
    index=pd.date_range("2021-01-01", periods=100, freq="h"),
)
predictions = estimator.predict(X_pred)

# Convert back to original scale if log-transformed
predictions_original = np.exp(predictions)
```
